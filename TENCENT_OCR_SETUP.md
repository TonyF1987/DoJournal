# 腾讯云OCR配置指南

## 概述

本项目使用微信云开发的OCR服务进行图片文字识别。这是微信官方提供的OCR服务,通过云调用方式使用,无需额外的API Key配置。

## 微信OCR优势

- **无需额外配置**: 通过云调用直接使用,无需申请API Key
- **免费使用**: 对小程序开发者免费
- **官方支持**: 微信官方提供的服务,稳定性好
- **简单集成**: 开箱即用,只需配置权限

## 配置步骤

### 1. 确认云开发环境已开通

在微信开发者工具中:
1. 点击顶部菜单"云开发"
2. 如果提示"云开发未开通",点击"开通"
3. 按照提示完成开通流程

### 2. 启用OCR服务

微信OCR服务是微信云开发的内置能力,无需单独开通。但需要确保:
- 云开发环境已创建
- 云函数已正确部署

### 3. 配置云调用权限

在微信开发者工具中:
1. 右键点击 `cloudfunctions/ocrGeneral` 文件夹
2. 选择"上传并部署:云端安装依赖"
3. 等待部署完成

**重要**: `config.json` 已正确配置了权限:
```json
{
  "permissions": {
    "openapi": [
      "ocr.printedText"
    ]
  }
}
```

### 4. 重新部署云函数

1. 在微信开发者工具中
2. 右键点击 `cloudfunctions/ocrGeneral` 文件夹
3. 选择"上传并部署:云端安装依赖"
4. 等待部署完成(约30秒)

## 测试OCR功能

部署完成后,测试一下功能:

1. 打开小程序
2. 进入"添加作业"页面
3. 点击"从聊天导入"或选择图片
4. 上传包含文字的图片
5. 观察是否能正常识别并填充内容

## 常见错误及解决方案

### 错误1: `errCode: -604100 API not found`

**原因**: OCR服务未启用或权限配置不正确

**解决方法**:
1. 确认云开发环境已开通
2. 检查 `config.json` 中的权限配置是否为 `ocr.printedText`
3. 重新部署云函数
4. 等待10分钟(权限配置有10分钟缓存)

### 错误2: `errCode: 0` 但返回空结果

**原因**: 图片无法识别或图片不清晰

**解决方法**:
1. 确保图片清晰,文字可见
2. 图片不要太小,建议宽度 > 500px
3. 文字尽量横向排列,不要歪斜
4. 使用拍照时确保对焦清晰

### 错误3: 云函数调用失败

**原因**: 云函数未部署或部署失败

**解决方法**:
1. 检查云函数是否已部署成功
2. 查看云函数日志确认错误信息
3. 重新上传并部署云函数

### 错误4: `errCode: -1` 提示"权限不足"

**原因**: 云调用权限未正确配置

**解决方法**:
1. 检查 `config.json` 中的权限配置
2. 确认格式正确:
```json
{
  "permissions": {
    "openapi": [
      "ocr.printedText"
    ]
  }
}
```
3. 重新部署云函数
4. 等待10分钟让权限生效

## API参数说明

### 云函数调用参数

```javascript
wx.cloud.callFunction({
  name: 'ocrGeneral',
  data: {
    imgUrl: 'cloud://xxx-xxx.xxx/xxx.jpg'  // 云存储文件ID或临时URL
  }
})
```

### OCR接口参数

```javascript
await cloud.openapi.ocr.printedText({
  img: base64Image    // base64编码的图片数据
})
```

**参数说明**:
- `img`: base64编码的图片数据
  - 必须是纯base64字符串,不能包含 `data:image/jpeg;base64,` 前缀
  - 支持的图片格式: JPG、PNG等
  - 图片大小限制: 最大不超过2MB
  - 云函数会自动将云存储URL或临时URL转换为base64

### 返回数据格式

```javascript
{
  errCode: 0,
  errMsg: 'ok',
  items: [
    {
      text: '识别出的文字',
      pos: {
        left_top: { x: 10, y: 20 },
        right_top: { x: 100, y: 20 },
        right_bottom: { x: 100, y: 50 },
        left_bottom: { x: 10, y: 50 }
      }
    }
  ]
}
```

## 性能优化建议

1. **图片压缩**: 上传前压缩图片可以减少流量
2. **缓存结果**: 如果同一图片识别多次,可以缓存结果
3. **异步处理**: 使用loading提示用户识别中

## 识别效果提升建议

### 拍摄时注意

1. **光线充足**: 确保拍摄环境光线良好
2. **对焦清晰**: 确保文字清晰可见
3. **避免反光**: 避免玻璃反光或阴影
4. **横向排列**: 文字尽量横向排列,不要歪斜

### 图片处理

1. **裁剪无关区域**: 只保留文字部分
2. **调整对比度**: 增加对比度可以提高识别率
3. **避免模糊**: 模糊的图片识别率会降低

## 限制说明

### 调用频率限制

- 每个云开发环境无明确调用次数限制
- 但建议合理使用,避免滥用

### 图片大小限制

- 图片大小建议不超过 10MB
- 图片宽度建议在 500-3000px 之间
- 支持的图片格式: JPG, PNG

### 识别内容限制

- 支持中文、英文、数字
- 支持印刷体文字
- 不支持手写体识别
- 不支持表格识别

## 其他OCR服务对比

如果微信OCR不能满足需求,可以考虑:

### 百度OCR
- **优势**: 免费额度500次/天,识别准确率高
- **缺点**: 需要申请API Key,需要配置环境变量
- **适用**: 需要更高准确率的场景

### 阿里云OCR
- **优势**: 免费额度500次/天,支持手写体
- **缺点**: 需要申请API Key,需要配置环境变量
- **适用**: 需要识别手写体的场景

目前微信OCR是最简单的选择,推荐优先使用。

## 注意事项

1. **缓存时间**: 权限配置有10分钟缓存时间
2. **环境要求**: 必须使用云开发环境
3. **权限配置**: 必须在 `config.json` 中配置正确权限
4. **重新部署**: 修改代码后必须重新部署云函数
5. **日志查看**: 出现问题时优先查看云函数日志

## 故障排查流程

当OCR功能不工作时,按以下顺序检查:

1. ✅ 检查云开发环境是否已开通
2. ✅ 检查云函数是否已部署成功
3. ✅ 检查 `config.json` 权限配置是否正确
4. ✅ 检查图片是否清晰可识别
5. ✅ 查看云函数日志获取详细错误信息
6. ✅ 尝试重新部署云函数
7. ✅ 等待10分钟后重试(权限缓存)

如果以上步骤都无法解决问题,建议:
- 查看详细的错误日志
- 检查云开发控制台的状态
- 联系微信云开发技术支持
