# 阿里云OCR快速开通指南

## 错误说明

遇到错误:
```
InvalidApi.NotPurchase: code: 403
```

**原因**: 阿里云OCR服务未开通,需要先在控制台开通服务。

## 快速开通步骤(5分钟完成)

### 步骤1: 登录阿里云控制台

1. 访问 https://www.aliyun.com/
2. 点击"登录",使用手机号/邮箱登录
3. 如果没有账号,先点击"免费注册"

### 步骤2: 完成实名认证

如果提示需要实名认证:

1. 点击右上角头像 → "实名认证"
2. 选择"个人认证"
3. 上传身份证正反面照片
4. 进行人脸识别
5. 认证通常在5-10分钟内完成

### 步骤3: 开通文字识别服务

#### 方法1: 使用免费额度(推荐)

1. 在阿里云控制台搜索框输入"文字识别"
2. 点击搜索结果中的"文字识别"
3. 进入"通用文字识别"产品页面
4. 点击"立即开通"或"免费试用"
5. 选择"免费试用"套餐
6. 点击"立即开通"
7. 开通成功后会显示"已开通"状态

**免费套餐包含**:
- 通用文字识别: 1000次/月
- 通用文字识别(高精版): 50次/月
- 身份证识别: 100次/月
- 其他识别服务: 各100次/月

#### 方法2: 按量付费(适合高频使用)

1. 进入"文字识别"产品页面
2. 选择"通用文字识别"
3. 点击"立即开通"
4. 选择"按量付费"套餐
5. 阅读《文字识别服务协议》
6. 点击"确认开通"

**按量付费价格**:
- 通用文字识别: 约0.001元/次
- 通用文字识别(高精版): 约0.002元/次
- 可以设置单日上限,避免超额

### 步骤4: 获取AccessKey

#### 创建RAM子账号(推荐,更安全)

1. 进入"访问控制" → "用户"
2. 点击"创建用户"
3. 填写信息:
   - 用户名: `ocr-miniprogram` (可自定义)
   - 访问方式: 勾选"编程访问"
4. 点击"确定"
5. 创建成功后,会显示 **AccessKey ID** 和 **AccessKey Secret**
6. **重要**: 请立即保存这两个密钥,刷新页面后将无法再次查看

#### 为RAM用户添加权限

1. 在"用户"页面,找到刚创建的用户 `ocr-miniprogram`
2. 点击用户名称进入详情页
3. 点击"添加权限"
4. 在搜索框输入 `AliyunOCRFullAccess`
5. 点击权限名称,点击"确定"
6. 权限添加成功

### 步骤5: 配置云函数环境变量

#### 在微信开发者工具中配置

1. 点击顶部菜单"云开发"
2. 进入"云函数"管理
3. 找到 `ocrGeneral` 云函数
4. 点击进入详情页
5. 点击"配置" → "环境变量"
6. 添加以下两个环境变量:

| 变量名 | 变量值 |
|--------|--------|
| `ALIYUN_OCR_ACCESS_KEY_ID` | 你的AccessKey ID (如: LTAI5t...) |
| `ALIYUN_OCR_ACCESS_KEY_SECRET` | 你的AccessKey Secret (如: xxxxxxxx) |

7. 点击"保存"

### 步骤6: 重新部署云函数

1. 返回微信开发者工具
2. 右键点击 `cloudfunctions/ocrGeneral` 文件夹
3. 选择"上传并部署:云端安装依赖"
4. 等待部署完成(约1-2分钟)

### 步骤7: 测试OCR功能

1. 打开小程序
2. 进入"添加作业"页面
3. 选择一张清晰的图片(包含文字)
4. 观察识别结果

## 常见问题

### Q1: 提示"账号未实名认证"

**解决方法**:
1. 进入账号中心完成实名认证
2. 上传身份证照片
3. 完成人脸识别
4. 等待认证通过(通常5-10分钟)

### Q2: 提示"服务未开通"

**解决方法**:
1. 进入"文字识别"产品页面
2. 点击"立即开通"或"免费试用"
3. 选择免费套餐
4. 点击"立即开通"

### Q3: 找不到"文字识别"产品

**解决方法**:
1. 在控制台顶部搜索框输入"文字识别"
2. 点击搜索结果进入产品页面
3. 或直接访问: https://www.aliyun.com/product/ocr

### Q4: 免费试用提示"已试用过"

**解决方法**:
1. 选择"按量付费"套餐
2. 按量付费也很便宜(约0.001元/次)
3. 可以设置日上限,避免超额

### Q5: 开通后仍然报错403

**可能原因**:
1. 服务刚开通,需要等待5-10分钟生效
2. RAM用户权限未正确添加
3. AccessKey配置错误

**解决方法**:
1. 等待10分钟后重试
2. 检查RAM用户是否有`AliyunOCRFullAccess`权限
3. 确认环境变量中的AccessKey正确

### Q6: AccessKey忘记了怎么办?

**解决方法**:
1. 进入"访问控制" → "用户"
2. 找到对应的用户
3. 点击"创建AccessKey"
4. 创建新的AccessKey
5. 旧AccessKey会自动失效

## 安全提醒

### ⚠️ 重要安全提示

1. **不要泄露AccessKey**: AccessKey等同于账号密码,泄露后会被滥用
2. **使用RAM子账号**: 不要使用主账号AccessKey
3. **最小权限原则**: 只授予必需的权限
4. **定期更换密钥**: 建议每3-6个月更换一次
5. **监控使用情况**: 定期查看调用日志,发现异常立即更换密钥

### 密钥安全检查清单

- [ ] AccessKey已妥善保存
- [ ] 没有将密钥提交到代码仓库
- [ ] 没有将密钥发送给他人
- [ ] 使用RAM子账号,不是主账号
- [ ] RAM用户只授予了OCR相关权限
- [ ] 定期检查调用日志

## 快速故障排查

### 错误1: `InvalidApi.NotPurchase: code: 403`

**原因**: OCR服务未开通

**解决**: 按照本文档步骤1-3开通服务

### 错误2: `Forbidden.RAM: User not authorized`

**原因**: RAM用户没有权限

**解决**: 为用户添加`AliyunOCRFullAccess`权限

### 错误3: `InvalidAccessKeyId.NotFound`

**原因**: AccessKey ID不存在

**解决**: 检查环境变量中的AccessKey ID是否正确

### 错误4: `SignatureDoesNotMatch`

**原因**: AccessKey Secret错误

**解决**: 检查环境变量中的AccessKey Secret是否正确

### 错误5: `QuotaExceeded: Daily quota exceeded`

**原因**: 日配额已用完

**解决**: 等待第二天或增加配额
